# Expense Tracker — Система учёта расходов

Веб-приложение для учёта личных расходов по категориям. Позволяет создавать категории трат, добавлять расходы, фильтровать и искать их.

---

## Архитектура

Выбран **Вариант Б — Fullstack Next.js**.

Фронтенд и бэкенд объединены в одном Next.js-приложении. Серверная логика реализована через Route Handlers в директории `app/api/`. Данные хранятся в памяти процесса (`lib/store.ts`). Клиентские компоненты обращаются к роутам через `fetch`.

**Обоснование выбора:** приложение небольшое, один разработчик, нет нужды в отдельном Express-сервере и настройке CORS. Вариант Б проще в развёртывании и разработке: один проект, один порт, меньше конфигурации.

## Структура данных

### Category (Категория)

| Поле | Тип | Обязательное | Описание |
|------|-----|:---:|---------|
| `id` | `string` | ✓ | Уникальный идентификатор (`cat-<timestamp>`) |
| `name` | `string` | ✓ | Название категории (уникальное, мин. 2 символа) |
| `color` | `string` | ✓ | Цвет в формате HEX |
| `isActive` | `boolean` | ✓ | Активна ли категория |
| `description` | `string` | — | Описание категории |

### Expense (Расход)

| Поле | Тип | Обязательное | Описание |
|------|-----|:---:|---------|
| `id` | `string` | ✓ | Уникальный идентификатор (`exp-<timestamp>`) |
| `title` | `string` | ✓ | Название расхода (мин. 2 символа) |
| `amount` | `number` | ✓ | Сумма (положительное число) |
| `date` | `string` | ✓ | Дата в формате `YYYY-MM-DD` |
| `categoryId` | `string` | ✓ | ID связанной категории |
| `note` | `string` | — | Примечание |

**Связь:** одна категория — много расходов (один-ко-многим). Расход хранит `categoryId`, ссылающийся на `Category.id`.

## API

Базовый URL: `http://localhost:3000/api`

### Категории

#### `GET /api/categories`

Получение списка категорий с пагинацией.

Параметры: `?page=1&limit=10`

```json
{
  "items": [
    { "id": "cat-1", "name": "Продукты", "color": "#22c55e", "isActive": true, "description": "Еда и напитки" }
  ],
  "total": 8,
  "page": 1,
  "pages": 1
}
```

#### `GET /api/categories/:id`

Получение категории по ID вместе с её расходами.

```json
{
  "id": "cat-1",
  "name": "Продукты",
  "color": "#22c55e",
  "isActive": true,
  "expenses": [
    { "id": "exp-1", "title": "Пицца", "amount": 1200, "date": "2025-01-10", "categoryId": "cat-1" }
  ]
}
```

#### `POST /api/categories`

Создание категории.

Тело запроса:
```json
{ "name": "Спорт", "color": "#10b981", "isActive": true, "description": "Абонементы и инвентарь" }
```

Ответ `201`:
```json
{ "id": "cat-1750000000000", "name": "Спорт", "color": "#10b981", "isActive": true, "description": "Абонементы и инвентарь" }
```

Ошибка `422` — если категория с таким именем уже существует:
```json
{ "error": "Категория с таким названием уже существует" }
```

#### `PATCH /api/categories/:id`

Частичное обновление категории. Тело — любые поля из `POST`.

Ответ `200`: обновлённый объект категории.

#### `DELETE /api/categories/:id`

Удаление категории.

Ответ `200`:
```json
{ "message": "Категория удалена" }
```

### Расходы

#### `GET /api/expenses`

Получение списка расходов с пагинацией и фильтрацией. Каждый расход включает вложенный объект `category`.

Параметры: `?page=1&limit=10&categoryId=cat-1`

```json
{
  "items": [
    {
      "id": "exp-1", "title": "Пицца", "amount": 1200, "date": "2025-01-10",
      "categoryId": "cat-1", "note": "Доминос",
      "category": { "id": "cat-1", "name": "Продукты", "color": "#22c55e", "isActive": true }
    }
  ],
  "total": 22,
  "page": 1,
  "pages": 5
}
```

#### `GET /api/expenses/search`

Поиск расходов по названию с опциональной фильтрацией по категории.

Параметры: `?q=такси&categoryId=cat-2`

```json
{
  "items": [ { "id": "exp-3", "title": "Такси", "amount": 600, "date": "2025-01-12", "categoryId": "cat-2", "category": { ... } } ],
  "total": 1,
  "page": 1,
  "pages": 1
}
```

#### `GET /api/expenses/:id`

Получение расхода по ID с вложенной категорией.

```json
{
  "id": "exp-1", "title": "Пицца", "amount": 1200, "date": "2025-01-10",
  "categoryId": "cat-1", "note": "Доминос",
  "category": { "id": "cat-1", "name": "Продукты", "color": "#22c55e", "isActive": true }
}
```

#### `POST /api/expenses`

Создание расхода.

Тело запроса:
```json
{ "title": "Кофе", "amount": 250, "date": "2025-06-20", "categoryId": "cat-1", "note": "Двойной эспрессо" }
```

Ответ `201`: созданный объект расхода.

#### `PATCH /api/expenses/:id`

Частичное обновление расхода. Тело — любые поля из `POST`.

Ответ `200`: обновлённый объект расхода.

#### `DELETE /api/expenses/:id`

Удаление расхода.

Ответ `200`:
```json
{ "message": "Расход удалён" }
```

### Коды ошибок

| Код | Причина |
|-----|---------|
| `404` | Запись не найдена |

| `500` | Внутренняя ошибка сервера |
